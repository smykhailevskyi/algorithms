Задачі на роботу з графами є доволі поширеними на LeetCode або подібних платформах. Також вони часто зустрічаються на співбесідах.  
Сьогодні поговоримо, як ми можемо представити граф у коді і який спосіб краще вибрати в процесі вирішення задачі.

Є чотири основних способів зберігання графів 

<strong>Adjacency Matrix</strong> - матриця суміжності  
<strong>Adjacency List</strong> - список суміжності  
<strong>Edge List</strong> - список ребер  
<strong>Incidence Matrix</strong> - матриця інцидентності  

<strong>Adjacency Matrix</strong>  
Для графа з n вершинами створюється матриця <code>n * n</code> (<code>matrix[i][j]</code>)   
```text   
    0 1 2 3
0 [ 0 1 0 1 ]
1 [ 1 0 1 0 ]
2 [ 0 1 0 1 ]
3 [ 1 0 1 0 ]
```
Де: 1 - є ребро, 0 - немає ребра  

Якщо зважений граф (weighted graph):
```text
    0 1 2
0 [ 0 5 0 ]
1 [ 5 0 2 ]
2 [ 0 2 0 ]
```
то число - вага ребра

Представлення в <strong>С++</strong>
```C++
vector<vector<int>> graph(n, vector<int>(n, 0));
```
<strong>Складність:</strong>  
пам'ять - <code>O(V²)</code>   
перевірити наявність ребра - <code>O(1)</code>    
пройти всіх сусідів вершини - <code>O(V)</code>  

<strong>Використання</strong>  
Використовується тоді, коли потрібно дуже швидко перевіряти, чи існує ребро між двома вершинами:
1. <strong>Перевірка наявності ребра</strong> - <code>O(1)</code>. Це головна перевага матриці суміжності проміж інших способів.
```C++
if (graph[u][v]) {
    // ребро існує
}
```
2. <strong>Щільні графи (dense graph)</strong> - якщо кожна вершина з'єднана майже з кожною.  
Наприклад, маємо 1000 вершин ≈ 500 000 ребер. У такому випадку матриця займає прийнятний обсяг пам'яті, а доступ дуже швидкий.

3. <strong>Алгоритм Дейкстри (класична реалізація)</strong>.  
Є дві реалізації Дейкстри:   
***Через матрицю суміжності*** (складність <code>O(V²)</code>)  
Саме таку реалізацію часто показують у підручниках.
```C++
vector<vector<int>> graph;
```
***Через список суміжності + priority_queue*** (складність <code>O((V + E) log V)</code>)   
Ця реалізація використовується значно частіше на практиці.
```C++
vector<vector<pair<int,int>>> graph;
priority_queue<...>
```
4. <strong>Алгоритм Флойда - Воршелла</strong>  
Для знаходження найкоротших шляхів між усіма парами вершин, де вхідні дані саме матриця:
```C++
dist[i][j]
```
5. <strong>Визначення транзитивного замикання</strong>   
Чи можна потрапити з A до B?  
Для відповіді на це питання саме зручно працювати з матрицею.

6. <strong>Маленькі графи</strong>, де n ≤ 500  
В цьому випадку простіше написати:
```C++
vector<vector<int>> graph(n, vector<int>(n));
```
ніж будувати список суміжності.
 
<strong>Adjacency List</strong>  
Зберігають списки суміжних вершин  
```text
0 -> 1, 3
1 -> 0, 2
2 -> 1
3 -> 0
```
Реалізація на <strong>C++</strong>  

```C++
vector<vector<int>> graph = {
    {1, 3}, // сусіди вершини 0
    {0, 2}, // сусіди вершини 1
    {1},    // сусіди вершини 2
    {0}     // сусіди вершини 3
};
```
Якщо зважений граф, то замість номера вершини зберігають пару ***(вершина, вага)***:   
```text 
0 -> (1,5), (3,2)
1 -> (0,5), (2,7)
2 -> (1,7)
3 -> (0,2)
```
Реалізація на <strong>C++</strong>

```C++
vector<vector<pair<int, int>>> graph = {
    {{1, 5}, {3, 2}},
    {{0, 5}, {2, 7}},
    {{1, 7}},
    {{0, 2}}
};
```
Тут ***pair.first*** - номер сусідньої вершини, а ***pair.second*** - вага ребра.

<strong>Складність:</strong>  
пам'ять - <code>O(V + E)</code>  
пройти сусідів - <code>O(degree(v))</code>  
перевірити, чи є ребро - <code>O(degree(v))</code>  

<strong>Використання</strong>  
Саме таке представлення найчастіше використовується в задачах на LeetCode у більшості алгоритмів на великих розріджених графах:  
- BFS  
- DFS  
- Edsger W. Dijkstra (алгоритм Дейкстри)  
- Robert Tarjan (SCC)  
- MST  
- топологічне сортування
  
***Більшість графів розріджені, а список суміжності зберігає лише існуючі ребра.***

<strong>Edge List</strong>  
Зберігаються лише ребра:
```text
(0,1)
(0,3)
(1,2)
(2,3)
```
Реалізація на <strong>C++</strong>
```C++
vector<pair<int,int>> edges;
```
або
```C++
struct Edge {
    int from;
    int to;
};
```
Для ваг:
```C++
struct Edge {
    int from;
    int to;
    int weight;
};
```

<strong>Складність:</strong>  
пам'ять - <code>O(E)</code>  
знайти всіх сусідів - <code>O(E)</code>

<strong>Використання</strong>  
Використовується в алгоритмах, які працюють саме зі списком ребер, наприклад:  
- алгоритм Д. Крускала (Joseph Kruskal)
- алгоритм Беллмана - Форда (Richard Bellman / Lester Ford Jr.)

<strong>Incidence Matrix</strong>  
В цьому випадку рядки це вершини, а стовпці - ребра.
```text
    e1 e2 e3
v0  1  0  1
v1  1  1  0
v2  0  1  1
```

<strong>Складність:</strong>  
пам'ять - <code>O(V×E)</code>

<strong>Використання</strong>  
Використовується значно рідше, переважно в теорії графів та спеціалізованих задачах.


<em>Якщо підсумувати, то для більшості задач на LeetCode та алгоритмів найчастіше використовують **список суміжності (Adjacency List)**, тому що він займає мало пам'яті й дозволяє одразу отримує всіх сусідів потрібної вершини, не переглядаючи весь граф</em>.
